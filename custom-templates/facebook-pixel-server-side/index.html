Facebook Pixel - Server-Side - Custom Tag Template

Latest release: July 27, 2020 | Comments

The Facebook Pixel - Server-Side custom tag template is an unofficial tag template for Google Tag Manager’s community template gallery.

Resource
Vendor documentation
Blog post
Gallery entry
GitHub repo

PERMISSIONS

Description

This template implements the Facebook Pixel on the website.

Contrary to the Facebook Pixel tag template, it doesn’t load any third-party JavaScript.

It is designed to communicate with a Server container running a Facebook Client and Facebook templates. It will not work without these components in place.

Instructions

The tag operates in the same way as the Facebook Pixel template, so you should reference that document for most of the deployment instructions.

The new feature is the Server-Side API Configuration group, which includes settings specifically for enabling Facebook pixel tracking in your Server container.

GTM Server Endpoint

This is where you’d type the URL of your Server container. The template will dispatch the requests to this endpoint.

Store FBCLID In Cookie

In case the URL has the fbclid parameter, the template will store this in the first-party _fbc cookie. This is to attribute conversions to the correct Facebook ad click.

Opt Out (Ads Delivery Optimization)

If this is set to True, the hit will only be used for attributing conversions to ads, and not for personalization or for augmenting the user’s tracking profile.

Source URL

You can set this field to have the hit register to a different URL than the one the user is currently on.

Event ID

If you are collecting events with both Facebook’s web SDK and this server-side setup, you can use the Event ID to deduplicate between events that might be sent from both sources.

Test ID

You can set this if you want to test your setup. The Test Event Code is provided by Facebook when you create the pixel in Events Manager.
